Confirmed Fix

ERSC no longer hangs after Golden Pot.

The 2026-05-16 fix disables Wine esync for the Elden Ring ERSC profile so Steam, Spacewar/AppID 480, and ERSC stop exhausting pipe and eventfd descriptors.

Symptoms

ERSC launched and entered the game, but opening the Golden Pot lobby could hang or close the game after a few minutes. Activity Monitor could show high Metal and app memory numbers, but no new macOS memory-pressure report was produced.

Log Signature

gptk: open-file limit 49152
pipe: Too many open files
eventfd: Too many open files
NtCreateFile Too many open files
wine: Unhandled page fault

Fix

The Elden Ring ERSC profile now repairs itself to noEsync = true. The launcher starts both Steam and ERSC with GPTK_WINEESYNC=0.

Stop any older Wine session before testing:

gptk-steam --kill

Result

After disabling esync for the ERSC profile and restarting the Wine prefix, the Golden Pot workflow no longer hung.
